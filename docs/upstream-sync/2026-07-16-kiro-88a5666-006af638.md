## General upstream feature decisions

- Integrate endpoint mode `auto` as an opt-in capability. Preserve `q` as the
  default, preserve account-level runtime configuration, and keep API Key
  accounts on Q.
- Exclude the four approved scope-expansion groups: upstream billing-rate
  introspection/probe/scheduling; OpenAI/Codex reliability fixes; image-input
  pricing, usage, and OAuth image usage; and Grok passive image intent and key
  templates.
- Automatically exclude later product-specific, non-general, operations, and
  test-convenience features from this KIRO sync only after documenting what
  each feature does.

These decisions apply only to the pinned audit range and do not replace the
normal Wei-Shaw upstream-sync process.
