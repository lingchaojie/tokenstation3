## 已确认的产品取舍

### 1. OpenAI 长上下文计费

- 已有 OpenAI parent 账号缺少字段时迁移为 `true`，显式布尔值原样保留，畸形值归一为 `false`；新建账号省略字段时为 `false`。
- 已有和未来 Spark shadow 都继承 parent 的有效值，parent 更新通过 scheduler outbox 传播。
- 该开关只决定用户账单；provider account cost 始终按真实长上下文价格计算，避免利润视图失真。
- usage log 保存当次布尔快照，历史账单不会因后续账号编辑而改变。

### 2. OpenAI Agent Identity

- `auth_mode=agentIdentity` 不要求也不伪造 OAuth access/refresh token 或 expiry。
- 已有 OAuth 账号切换为 Agent Identity 时，清除 access/refresh/id token、client、token expiry 和账号级 expiry，并关闭 OAuth 过期自动暂停；`model_mapping` 等本地路由元数据继续保留。
- 即使历史脏数据仍残留 refresh token，刷新候选查询、后台 refresher、直接 OAuth refresh 入口和请求路径 token provider 都会拒绝 Agent Identity。
- task 注册复用账号代理；每次上游请求生成新 assertion；invalid/expired task 只恢复一次，并按 credential account 串行化注册。
- task 更新会使旧 WS 连接失效；Spark shadow 始终解析到 credential parent。
- 私钥、task ID、assertion 以及包含它们的错误体不会进入 API 响应、普通日志或 ops 事件。

### 3. Grok URL 与媒体路由

- 默认 Grok OAuth 文本/Responses 使用本地既有 CLI subscription gateway。
- 默认 OAuth image/video 使用官方 `api.x.ai`，规避 CLI gateway 较小的 body 限制。
- 只有显式配置且通过 URL policy 的 custom `base_url` 才统一接管文本、媒体、quota、billing、monitor 和 probe。
- OAuth 授权与 token refresh 无论是否 custom 都使用官方认证端点；header override 不能覆盖认证、连接、WS、framing 或 session-isolation header。

### 4. 管理员充值返佣

- 总开关默认 `false`；只接受正数 `add`，`set`、`subtract`、0 和负数均不返佣。
- 继续使用既有 eligibility、rate、freeze、duration 和 cap；用户首次充值的本地固定逻辑不被替换。
- 返佣是余额操作后的附加副作用，失败只记录已脱敏日志，不回滚成功的余额变更。

### 5. Server-Timing

- 默认关闭；关闭时 SQL/Redis 客户端不包装计时器。
- 请求头只能缩小采集范围，不能获得权限；只有已认证 admin/user UI 响应可见指标。
- 指标有数量边界，只含 duration/count，不含 SQL、Redis key、凭据、请求体或 gateway 数据。
