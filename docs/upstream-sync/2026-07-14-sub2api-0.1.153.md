## KIRO 专项复核

依照 `docs/kiro-upstream-sync.md` 对 KIRO 专属与共享路径进行了复核：

- KIRO direct 与 relay 账号仍分别进入原生 runtime 和共享 Anthropic-compatible 路径；
- sticky/session、token refresh、auth cache invalidation、custom upstream headers、cooldown、usage/credits 与 WebChat capture 仍保留；
- OpenAI-compatible usage 合并后仍传播 `KiroCredits`，没有被上游较窄 usage 类型覆盖；
- Q chat、Q MCP、KRS 的 profile ARN 放置与 machine ID 持久化逻辑未被改动；
- 前端 KIRO 平台入口、账户工作流、状态/用量展示、模型映射和 group 设置未被上游平台列表覆盖。

本轮没有升级 `nianzs/sub2api` KIRO 参考点，`docs/kiro-upstream-sync.md` 继续固定 `88a5666b478e234cace9090e0d5f483f1146cb96`。
