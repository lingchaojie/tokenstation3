## 独立 review 发现与修复闭环

首次独立全量 review 的 3 个 reviewer 均逐文件覆盖分配范围，并报告 4 个可执行问题；在修复前结论均为 `NOT SAFE`，因此没有推送 dev。

| 发现 | 修复与证明 |
|---|---|
| Fast Policy 无效 `user_ids` 在通用设置写入后才报错，可能部分保存 | 后端把 policy 校验前移到任何写入之前；前端同时校验正整数与唯一性且禁止静默过滤；增加 handler 零写入测试和 SettingsView 零值/重复/合法提交测试 |
| WS v2 passthrough 未识别 Windows TCP reset 文案 | 同步两种 Windows 断线短语，使其进入 graceful drain；增加两条回归测试 |
| 嵌套 keepalive writer 仍可能持有已放回 pool 的 ops capture writer | Ops middleware 返回前无条件恢复原始 writer，再释放 capture writer；增加嵌套 wrapper 回归并用 `-race` 验证 |
| 同名 `custom` / `function` 工具回程会被误分类 | 请求转换阶段显式拒绝跨类型同名顶层工具；覆盖两种声明顺序及 custom/function tool choice |
| Messages 流式 fallback 被窄 `ChatUsage` 覆盖后丢 cache aliases 与 `KiroCredits` | 以 raw CC usage accumulator 作为终态 wire 与内部计费的共同权威来源；合法全零与 usage 缺席分离 |
| namespace 强制 `tool_choice` 被静默降级 | 单子工具 namespace 精确摊平；零子或多子无法无损表达时显式拒绝；普通与流式回程恢复 namespace |
| CC/Responses 双方言、malformed/partial usage 会错桶、清零或导致 fallback 502 | usage 与内容解码彻底隔离；CC-first、有效非负数和字段级 presence 累计覆盖 cache/image/audio/reasoning/prediction/Kiro，非流式与流式 wire 和 billing 统一 |
| CC→Responses 投影丢 cache-write 与 image 明细 | 补齐 cache write、image token 和完整 output detail 双向转换，并保留 canonical 显式零 |
| Embeddings 自动合入了 Responses-first cache helper | 改为 Embeddings 专用 prompt-first、presence-aware parser，显式零优先且非法/负数回退下一别名，避免 cache read/write 错桶 |
